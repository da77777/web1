HTML

html의 구조

  1. title : 주소창 위 탭에 표시되는 웹페이지의 타이틀.
  2. meta : meta charset="utf-8"의 구성으로 사용한다.
    char은 문자, set은 규칙이라는 뜻이며
    utf-8로 문서를 읽으라 즉, 이 문서는 utf-8 방식으로 저장되어 있다는 뜻이다.
  3. head : title, meta와 본문을 구분하기 위해 구성에 삽입하는 필수 태그.
  4. body : title, meta와 본문을 구분하기 위해 본문에 삽입하는 필수 태그.
  5. html : head와 body를 감싸는 태그.
  6. !DOCTYPE html : html 위에 관용적으로 작성하는 태그. 이 문서는 html이라는 뜻이다. 닫는 태그는 없다.

※ html의 뼈대
< !DOCTYPE html>
< html>

< head>
< title>
< meta charset="">
< /head>

< body>
< /body>

< /html>

태그

  1. strong : 굵게
  2. u : 밑줄(underline)
  3. hn : 제목(heading) 표시. 글자가 굵어지며 줄바꿈되는 효과가 있으며, h뒤의 숫자는 1~6까지 있고 숫자가 커질수록 글자의 크기는 작아진다.
  4. br : 줄바꿈. 닫는 태그는 없다.
  5. p : 단락 구분. 위아래 공백이 생긴다. 정보 제공으로서 적합한 태그는 br 보다는 p 태그이다.
    br은 단순 시각적인 줄바꿈인 반면, p태그는 주제에 따라 의미 있게 단락을 구분하는 것이기 때문이다.
  6. img : 이미지삽입. img src="이미지이름" width="n" 의 구성으로 사용한다.
    이미지 다운로드하고 web 폴더로 이동 후 사용하면 된다.
    img에는 src와 width라는 2가지 속성(attribute)이 추가되며, 각 소스 작성 순서는 상관 없다.
    src는 '소스' 의 약자이며 삽입하고자 하는 이미지의 이름을 기재하면 된다.
    width는 이미지의 크기를 조절해주는데, width="100"과 width="100%"는 다르다. 100%의 경우 화면 크기에 따라 이미지 크기도 자동으로 변경된다.
  7. a : 링크태그.
    a href="링크 주소" 구성으로 사용하며, h는 하이퍼텍스트, ref는 참조를 뜻한다.
    클릭 시 새 탭을 열어 보여주고 싶으면 target="_blank"를 사용하며
    마우스를 올려놓았을 때 링크 이름이 나오게 하고싶은 경우 title="링크 이름"을 사용한다.
    최종적으로는 a href="링크 주소" tareget="_blank" title="링크 이름">의 구성이 된다.
  8. 부모자식관계의 태그 : ul/ol - li
    • li : 목차 구성시 사용하는 태그로 list의 약자.
    • ul : 목차를 구분해주는 태그로 unorderd list의 약자. li 사용 시 ul은 필수 사용 태그이다.
      가령 목차1은 HTML, CSS, JavaScript를 나타내고, 목차2는 참가자1, 참가자2, 참가자3을 의미할 때, 목차1과 목차2를 의미적으로 구분해주는 태그이다.
    • ol : 목차를 구분해주는 태그로, orderd list의 약자. 전반적으로 ul과 동일하다.
      차이점은 ul의 경우 목차에 일일이 순번을 기재해야하는 반면, ol의 경우 알아서 순번을 기재해준다.
      목차가 여러개인 경우 ol이 유용하다. 가령 목차가 20개인데 1번 목차를 삭제해야 하는 경우, ul은 손으로 수정해야 하는데 ol은 1번 목차를 삭제하면 자동으로 숫자가 바뀐다.
      이 페이지의 오픈소스에서 하단 예시 중 ol태그의 목차를 아무거나 지워보자.
      • 예시-ul 사용 시
        • 1. HTML
        • 2. CSS
        • 3. JavaScript
      • 예시-ol 사용 시
        1. HTML
        2. CSS
        3. JavaScript
